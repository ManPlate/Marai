# Marai — Developer Guide

## Requirements

- Python 3.10 or higher
- pip packages: `cryptography`, `pyperclip`

## Setup

```bash
pip install -r requirements.txt
python marai.py
```

## Building the .exe

Use the included build script:

```
build.bat
```

This will:
1. Read the version from `marai.py`
2. Write `version.json`
3. Update the version table in `README.md`
4. Clean old build files
5. Build `Marai.exe` using PyInstaller

Output: `dist\Marai.exe`

## File Structure

```
Marai/
├── marai.py           ← main app (update this on each release)
├── write_version.py   ← helper called by build.bat
├── build.bat          ← build script
├── requirements.txt   ← Python dependencies
├── version.json       ← auto-generated by build.bat
├── README.md          ← GitHub homepage
├── README_DEV.md      ← this file
├── HOW_TO_UPDATE.md   ← end user update guide
├── run_windows.bat    ← run from source on Windows
└── run_mac_linux.sh   ← run from source on Mac/Linux
```

## Data Storage

Vault data is stored locally at:
- Windows: `C:\Users\YourName\.marai\`
- Mac/Linux: `~/.marai/`

Files:
- `vault.enc` — AES-256-GCM encrypted password entries
- `meta.json` — salt and verification token

## Release Process

1. Make changes in `marai.py`
2. Update `VERSION = "x.x.x"` at the top
3. Add a line to `CHANGELOG` describing the change
4. Run `build.bat`
5. Upload `marai.py`, `version.json`, and `README.md` to GitHub
6. Create a new GitHub Release with the new version tag
7. Attach `dist\Marai.exe` to the release
